‚öñÔ∏èLOAD BALANCING (ELB)
üéõÔ∏èCore Features
Distribute traffic across targets across multi-AZs
Health checks for availability (HTTP 200 OK)
Auto-scaling, high availability
Rules = by priority [conditions + actions]
Rules allow creating highly customizable traffic routing strategies
Rule types: Listener, target group, host/path/query-string based
üìãELB Rules & Algorithms
Listener rules: Protocol/port ‚Üí Target group
Target group rules: Distribution algorithm
Priorities: Lower number = higher precedence
Algorithms
Round robin: Sequential
Least connections: Least-loaded
IP hash: Session persistence
üîßLoad Balancer Types
ALB: advanced routing capabilities for modern web applications
Protocols: Layer 7 HTTP/HTTPS
Advanced routing: URL paths, hostnames, query strings, HTTP headers, methods, and source IPs
Request-based routing: Each HTTP request can be routed independently based on rules (opposite of NLB which is flow-based)
SSL/TLS termination
Sticky sessions
Targets: EC2, IP, Lambda, containers
SNI (Server Name Indication) support for multiple HTTPS domains on the same ALB without using multiple IPs
Dynamic port mapping
NLB: Limited to high-speed routing cases, analyzing IP data not headers or content of the request
Protocols: Layer 4 TCP/UDP ‚Äì versatile for apps with varying protocol needs
Ultra-low latency (<100 Œºs) & Millions req/sec
Health checks: ICMP pings or 3-way TCP handshake
Static IPs, preserve source IP
Flow-based load balancing: maintains session (~flow~connection) affinity at the connection level: Once the NLB receives a flow, it maps that flow to a specific target and sends all packets in that flow to the same target.
Use: Gaming, IoT, real-time streaming, high-throughput
GWLB: optimized for virtual network appliances
Protocols: Layer 3 IP (GENEVE) - packet contents
Deploy security appliances: Firewalls, IDS/IPS, NAT gateways
Centralized management of network security appliances
Use: Security, packet inspection, automatic scaling of virtual appliances
CLB (Classic Load Balancer) - DEPRECATED
Layer 4/7, basic routing
Round-robin (good for similar capacity endpoints), least connections, IP hash (good for stateful applications & session persistence)
Less flexible for containers due to static port mapping
